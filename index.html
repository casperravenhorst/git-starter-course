Git starter course

Why use git?

  • Versioning
  • Traceable code history
  • Cleaner code
  • Distributed system

Basics

http://github.com/casperravenhorst/git-starter-course

Cloning

git clone <adres>

Copies a remote repository into the current directory - will make a subfolder git-starter-course in this case.

Committing

git status

Shows what is in the index, and what more files have changed in any way

git diff

Shows the difference between your latest commit and the current content of the source code files (ie what would be added running `git add .`)

git diff --staged

Shows the difference between what is in the index and the current content of the source code files (ie what would be added running `git add .`)

git add <filename>

Adds file to the index, ready to be committed

git add .

Adds all changed files (also moves moved files in the commit, and removes removed files)

git commit

Commits current state of index - in most cases launches some version of vim to enter commit message

Pushing

git push

Sends all your local commits to the remote repository

git remote

Shows information about what place is currently the remote (upstream) repository for your local repository

git pull

Updates your local repository with what is on the remote repository

Branching

git checkout <category/branch-name>

Switches your local repository over to another branch

git branch

Shows a list of all (local) branches

git branch <category/branch-name>

Makes a new branch, with the current branch as a starting point

git checkout -b <category/branch-name>

Makes a new branch, and switches to it immediately

git merge <category/branch-name>

Attempts to merge branch branch-name into the current branch

git branch -d <category/branch-name>

Deletes branch if it has been fully merged

git branch -D <category/branch-name>

Deletes branch, even if it has not been fully merged

Solving conflicts

Advice: use IDE, or other graphical tool to look at the diff
